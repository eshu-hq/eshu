## SSE Event Contract

Same route (`POST /api/v0/ask`), content-negotiated via `Accept: text/event-stream`.

Wire format per event:
```
event: <name>\n
data: <json>\n
\n
```

| Event name | Payload | Notes |
|------------|---------|-------|
| `trace` | `{"tool":"string","supported":bool,"truth_class":"string"}` | One per `AskAnswer.Trace` entry. Args omitted (may contain query params not explicitly authorised for streaming). |
| `answer` | Full `askResponse` JSON (same as JSON path 200) | One per request. |
| `error` | `{"state":"unavailable","reason":"string"}` | Emitted on engine error in place of answer+done. |
| `done` | `{}` | End-of-stream marker. |

Headers on SSE response: `Content-Type: text/event-stream`, `Cache-Control: no-cache`, `Connection: keep-alive`.

## Content Negotiation

`handleAsk` checks `acceptsSSE(r)` (detects `text/event-stream` in `Accept` header) and dispatches to `handleAskSSE` before any other logic. Non-SSE requests follow the unchanged JSON path.

## Default-off and Error Handling

- **nil Asker**: returns 503 JSON `{"state":"unavailable",...}` BEFORE opening the event stream (HTTP status 503 with `Content-Type: application/json`).
- **Empty/missing question**: returns 400 JSON before opening the stream.
- **No Flusher**: returns 500 JSON if the `ResponseWriter` does not implement `http.Flusher`.
- **Engine error**: SSE headers have already been committed (200); emits `event: error` with bounded `askUnavailableResponse`. Engine error text is never echoed.

## Leak Safety

Only bounded `traceEntry`, `askResponse`, and `askUnavailableResponse` values are emitted. Provider bodies, prompts, raw engine internals, and credentials are never written to the stream â€” the same contract as the JSON path.
