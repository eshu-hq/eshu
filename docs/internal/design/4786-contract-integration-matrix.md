# 4786 Contract-Integration Matrix

Issue #4786 audits the registry families and reducer-derived contracts that seed
the W1/W2 contract-integration wave under epic #4783. This is a read-only docs
artifact: it records current source truth and points the implementation issues at
their owning files. A matrix cell is either backed by a `file:line` citation or
is explicitly `none`.

## Scope

- Registry source: `specs/fact-kind-registry.v1.yaml:25` through
  `specs/fact-kind-registry.v1.yaml:405`, which currently covers 26 families.
- Reducer-derived source: ADR #4784 governs 22 reducer-derived kinds starting at
  `docs/internal/design/4784-reducer-derived-fact-governance.md:61` and leaves
  `reducer_cloud_asset_resolution` exempt until it gains a production consumer:
  `docs/internal/design/4784-reducer-derived-fact-governance.md:98`.
- Out of scope: reducer intent rows (`go/internal/storage/postgres/shared_intents.go:40`),
  shared followup trigger facts (`go/internal/reducer/shared_projection.go:246`),
  replay cassette and parser-fixture producers folded into #4797
  (`go/internal/replay/cassette/source.go:1`, `go/internal/replay/parserfixture/source.go:1`).

## Legend

Emit patterns are `inline-map`, `helper-map`, `struct-then-map`, `Encode`, or
`raw-SQL-writer`. Consumer seams are reported by layer as reducer, projector,
query, postgres loaders, relationships, and replay/offlinetier. Raw-read buckets
are `source-fact IN`, `intent-row OUT`, or `reducer-derived governed`.

## Special Notes

### Documentation Two-Struct Diff

The documentation family has two struct layers that are not identical:

- `go/internal/facts/documentation.go:118` defines emitter payload structs with
  concrete string/bool fields and `omitempty` tags for optional JSON output.
- `sdk/go/factschema/documentation/v1/document.go:18` defines schema structs
  where required fields are non-pointer and optional fields are pointers/slices.
- Required-field deltas are intentional: `Source` requires
  `source_id/source_system/external_id` at
  `sdk/go/factschema/documentation/v1/document.go:18`; `Document` requires only
  `document_id` at `sdk/go/factschema/documentation/v1/document.go:70`;
  `Section` requires `document_id/revision_id/section_id` at
  `sdk/go/factschema/documentation/v1/document.go:153`; `Link` requires
  `document_id/link_id/target_uri` at
  `sdk/go/factschema/documentation/v1/document.go:228`.
- `EntityMention` requires only the reducer identity fields
  `document_id/section_id/resolution_status` at
  `sdk/go/factschema/documentation/v1/entity_mention.go:25`; internal
  `DocumentationEntityMentionPayload` still emits `mention_id`, `mention_text`,
  and `mention_kind` as concrete strings at `go/internal/facts/documentation.go:184`.
- `ClaimCandidate` requires collector-always-set fields
  `document_id/section_id/claim_id/claim_type/claim_text/claim_hash/authority`
  at `sdk/go/factschema/documentation/v1/entity_mention.go:95`, while internal
  payloads remain concrete at `go/internal/facts/documentation.go:208`.
- `Finding` and `EvidencePacket` intentionally model only query-visible top
  level keys at `sdk/go/factschema/documentation/v1/finding.go:28` and
  `sdk/go/factschema/documentation/v1/finding.go:113`; nested
  `permissions/states/unified_evidence/finding` stay out because query reads
  them as raw SQL/map data at `sdk/go/factschema/documentation/v1/finding.go:14`.

### #4750 Landing Versus Residual parsed_file_data

#4750 landed as PR #4774 and added typed inner decoders for five
`parsed_file_data` keys: `gomod_state`, `function_calls_scip`,
`dockerfile_stages`, `pipeline_calls`, and `dead_code_file_root_kinds`

outer map: `sdk/go/factschema/codegraph/v1/file.go:61` still carries
`ParsedFileData map[string]any`, and residual raw reads remain in reducer
(`go/internal/reducer/code_import_repo_edge_identity.go:56`), relationships
(`go/internal/relationships/evidence.go:73`), and Postgres/query indexing
(`go/internal/storage/postgres/schema_fact_records.go:61`). Keep those files in
W2f/#4797/#4798 as listed below.
