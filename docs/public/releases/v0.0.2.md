# v0.0.2

Released: May 8, 2026

Source commit: `3a642b7c5da1eae9049c1f5b6369f03593ffaa7b`

## What Changed

This release is mostly about making Eshu's code-intelligence path more honest
and more useful.

Dead-code finding is now materially better for Go and the JavaScript family.
Eshu now treats common Go reachability patterns as roots, including function
values, dependency-injection callbacks, interface returns, method sets, and type
references. That closes the false positives we saw while dogfooding against
Eshu itself.

Node, JavaScript, TypeScript, and TSX also gained the first serious root model
for local service code. Package entrypoints, `bin` targets, package exports,
Next.js routes, Express registrations, configured Hapi/lib-api-hapi handler
exports, TypeScript interface implementation methods, and bounded static
module references are now handled as root evidence instead of dead-code
candidates.

The public claim is still intentionally conservative: dead-code results are
supported at derived truth, not cleanup-safe exact truth. The live MCP dogfood
matrix covered representative local Node/TypeScript services with
`find_dead_code` and targeted graph checks, but private repo names, paths, and
service identifiers stayed in local notes and are intentionally not committed.

The release also adds a dead-code fixture inventory for every parser-supported
language. Go has the active regression coverage in this release. Other language
fixtures now give us a place to mature each language in parallel without
pretending unsupported evidence is stronger than it is.

Local graph startup returns to the fast path. Setting `ESHU_NORNICDB_BINARY`
alone no longer pushes Eshu into slow process-mode NornicDB. Process mode now
requires `ESHU_NORNICDB_RUNTIME=process`, which keeps normal local-authoritative
runs on the expected embedded path.

MCP and API call guidance is now part of the repo. The new local skill and
agent instructions require scoped calls, limits, timeouts, truncation, cheap
preflights, and summary-before-payload behavior so future tool calls stay fast
and bounded.

Deployment defaults now point at `v0.0.2`. The Helm chart version is `0.0.2`,
the chart `appVersion` is `v0.0.2`, and the checked-in Helm, Argo CD, and
minimal manifest defaults use the new image tag.
