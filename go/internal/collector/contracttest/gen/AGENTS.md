# AGENTS.md - internal/collector/contracttest/gen guidance

## Read First
1. `README.md` — command purpose and usage.
2. `main.go` — the generator and `-check` logic.
3. `doc.go` — package contract.
4. `specs/collector_fact_contract.v1.yaml` — the source-of-truth spec.

## Invariants
- `run()` is the testable entry point; flag parsing is only in `parseOptions`.
- `-check` must never write files.
- Generated output must be deterministic — same YAML spec produces same bytes.
- Generated file starts with `// Code generated by ...` header.
- The command is invoked via shell wrappers (`scripts/generate-contracttest.sh`,
  `scripts/verify-contracttest.sh`).

## Common Changes
- Add a new collector to the YAML spec → the generator picks it up automatically.
- Change the generated Go code formatting → update `generateGo()`.
- Add validation to the spec loader → update `loadSpec()`.

## What Not To Change Without An ADR
- Do not add runtime imports (database, HTTP, queue).
- Do not change the generated file path or name without updating the shell
  wrappers and CI workflow.
- Do not change the `// Code generated by` header format — downstream tooling
  may depend on it.
